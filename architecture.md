# Архитектура: Структура файлов и проекта

## Тип проекта
Одностраничное веб-приложение. Без сервера, без базы данных.
Всё работает в одном HTML-файле — Саша открывает его на телефоне.

---

## Структура файлов

```
Radiator/
│
├── index.html          ← Главное приложение (интерфейс + логика в одном файле)
├── idea.md             ← Описание идеи
├── spec.md             ← Техническое задание
├── architecture.md     ← Этот файл
└── tasks.md            ← Список задач и прогресс
```

---

## Структура интерфейса (экраны)

```
┌─────────────────────────────┐
│           ШАПКА             │  «Быстрый расчёт отопления»
├─────────────────────────────┤

│     [РАССЧИТАТЬ] КНОПКА     │
├─────────────────────────────┤
│        КАРТОЧКА РЕЗУЛЬТАТА  │  Диапазон цены + разбивка
│  ┌─────────────────────┐    │

│  │ Материалы:    XX XXX│    │

│  │─────────────────────│    │
```

---

## JavaScript-функции (внутри index.html)

```
calculateCost(params)           ← Основная функция расчёта
  └─ applyMultipliers()         ← Применяет все коэффициенты
  └─ splitIntoCategories()      ← Разбивает итог на оборудование/материалы/монтаж

renderResult(result)            ← Обновляет DOM с результатами
generateShareText(result)       ← Формирует текст для отправки клиенту
```

---

## Поток данных

```
Пользователь заполняет форму
           ↓
     Нажимает «Рассчитать»
           ↓
  calculateCost(params)  →  объект с результатом
           ↓
  renderResult(result)   →  показывает карточку результата
           ↓
  [По желанию] generateShareText → отправить в WhatsApp / скопировать
```
